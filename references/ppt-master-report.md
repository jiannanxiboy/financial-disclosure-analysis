# PPT Master 财务报告契约

## 内容事实源

- 数值、比率和图表数据：`数据底稿.xlsx`
- 披露表述、原因和风险事实：官方 PDF/TXT 及 TSV 备注
- 推导结论：在 `报告素材.md` 中标注计算逻辑或证据链
- 设计与导出：PPT Master 项目及其当前 `SKILL.md`

报告素材不得把推测写成事实。原因无法由披露支持时，使用“可能”“需关注”并说明判断依据。

## 建议叙事结构

根据分析深度调整页数，不机械套版。标准报告通常包含：

1. 封面
2. 执行摘要：3–5 个结论
3. 核心指标全景
4. 收入规模与增长
5. 盈利能力与利润质量
6. 资产结构与资产质量
7. 杠杆、债务与流动性
8. 经营现金流与资金平衡
9. 业务分部或行业经营指标
10. 同行定位与综合评分
11. 风险、数据局限与关注事项
12. 附录：口径与关键来源

每页只承载一个主要判断。标题写成结论句，而不是“盈利分析”这类主题标签。

## 图表选择

- 期间趋势：折线图或斜率图
- 公司规模比较：排序条形图
- 构成变化：堆叠条形图；类别少且单期时才用环形图
- 收入/利润与现金流联动：组合图或小倍图
- 风险矩阵：二维散点或象限图
- 多指标同行画像：热力表；谨慎使用雷达图

优先让图表表达一个明确结论。不要为了“每页有图”制造无意义可视化。

## 财务演示设计约束

- 默认 16:9、中文管理层汇报风格、高信息密度但保持留白。
- 使用统一的正负/风险颜色语义；不要仅靠颜色传达关键信息。
- 对规模、比例、期间和币种做清晰标注。
- 页脚保留来源短注；完整定位放附录或 Excel 明细。
- 对关键图表优先使用 PPT Master 支持的数据驱动原生图表/表格导出选项；任何原生对象变体都要复核跨应用渲染差异。

## PPT Master 执行边界

1. 用桥接脚本创建项目并导入素材。
2. 读取上游 `SKILL.md` 和 routing authority，按当前版本选路。
3. 在 Strategist 强制确认门展示推荐的受众、页数、结构、风格、配色和图表策略，等待用户确认。
4. 确认后由主 Agent 严格按上游要求逐页手写 SVG；不得批量脚本生成页面，不得委派 SVG 页面生成。
5. 运行上游质量检查、`finalize_svg.py` 和 `svg_to_pptx.py`。
6. 正式交付 `exports/*.pptx`，保留 `svg_output/` 和 `svg_final/` 供追踪与预览。
7. 用 `validate_pptx.py` 核对 PPTX 页数、讲稿、图表、表格和画布；检测到 LibreOffice 时执行 PDF 往返渲染并核对页数。没有可用渲染器时记录 skipped，不以结构检查冒充跨应用渲染。

## 验收抽样

至少抽查：

- 封面后的执行摘要所有数字
- 每张图表的最大值、最小值和一个中间值
- 每个公司至少一个利润指标、一个资产负债指标和一个现金流指标
- 风险结论对应的来源或计算
- 所有页标题是否真正由页面内容支撑

发现数值差异时回到 Excel/TSV 修正，再更新 SVG 和 PPTX；不要直接在最终 PPTX 中手改后丢失证据链。
