# finscout — 财务报表分析与提取

## 适用平台

- **Claude Code**（原生）：克隆到 `.claude/skills/finscout/` 自动发现
- **其他 Agent**：将本文件作为 system prompt 注入，`scripts/` 加入 PATH 或 `cd` 到本目录执行

## 前置依赖

```bash
pip install -r requirements.txt
playwright install chromium
```

## 脚本清单

所有脚本位于 `<本skill目录>/scripts/` 下，Python 3.10+ 可直接运行：

| 脚本 | 用途 |
|------|------|
| `a_share.py` | A股年报搜索与PDF下载（巨潮资讯网） |
| `hk_share.py` | 港股年报搜索与PDF下载（港交所披露易） |
| `pdf_to_text.py` | 批量PDF转TXT（多进程） |
| `generate_excel.py` | 从TSV生成Excel数据底稿 |

## 阶段1 — 确认范围

**1a — 确认分析目标**：当用户未明确指定时，逐一询问确认：目标公司候选、时间范围、报表类型、输出目录。默认输出目录为当前工作目录下的 `finance_analysis/data`。

**1b — 判定分析深度**：用户请求中通常隐含了深度期望，按以下线索判定，无需额外询问：

| 用户措辞信号 | 深度 | 报告目标篇幅 | 指标维度覆盖 |
|---|---|---|---|
| "简单看看""快速过一下""大致比较" | 简要 | 3-5页 | 三张表核心科目及关键比率 |
| "分析""对比""经营情况和财务指标"（默认） | **标准** | **10页+** | 多角度覆盖，支撑深度对比分析 |
| "深度分析""尽调""全面评估""写份详细的" | 深度 | 15页+ | 在标准基础上进一步细化拆分 |

**用户未明确指定时，默认按"标准"深度执行**——即最终报告应达到10页以上篇幅。

**1c — 确认**：汇总目标公司、时间范围、报表类型、分析深度、输出目录，一次性请用户确认。这是唯一一轮确认，后续阶段自动推进。

## 阶段2 — 全量下载与提取

下载和PDF转txt是确定性脚本调用，由主线程直接执行。

### 阶段2a — 批量搜索并下载PDF（按市场分组）

用变量 `SD` 指向本 skill 的 `scripts/` 目录。

- **A股**：`python {SD}/a_share.py search-annual --codes {代码1} {代码2} ... --year {年份} --download-dir "{output_dir}/pdfs" --quiet`
- **港股**：`python {SD}/hk_share.py search-annual --codes "{代码1},{代码2},..." --year {年份} --download-dir "{output_dir}/pdfs" --quiet`

`--year` 仅接受单个年份。如需一次处理多个年份，使用 `--years`（A股空格分隔，港股逗号分隔）：

- **A股**：`python {SD}/a_share.py search-annual --codes {代码1} {代码2} ... --years {年1} {年2} {年3} -d "{output_dir}/pdfs" --quiet`
- **港股**：`python {SD}/hk_share.py search-annual --codes "{代码1},{代码2},..." --years {年1} {年2} {年3} -d "{output_dir}/pdfs" --quiet`

脚本内部复用浏览器、自动处理限流。默认使用 `--quiet`，只输出下载汇总和失败/未找到信息；需要核对具体公告标题或下载路径时再去掉 `--quiet`。

### 阶段2b — 批量转PDF为TXT（一条命令）

```bash
python {SD}/pdf_to_text.py --input-dir "{output_dir}/pdfs" --output-dir "{output_dir}/txt" --skip-existing --quiet
```

这条命令使用多进程并行处理所有PDF，无需逐文件调用。默认使用 `--quiet`，只输出汇总结果，避免第三方 PDF 解析库警告刷屏；如需排查具体 PDF 解析问题，可临时加 `--verbose` 查看详细日志。

### 阶段2c — 制定指标清单

用 Read 工具只读2-3份TXT的目录/摘要部分（每个市场至少一份），了解各市场实际报表结构和披露科目差异。

**指标设计方法**：

1. 从报表实际结构出发，列出所有可提取的科目和经营数据——先看有什么，再决定取什么
2. 指标清单应覆盖报表提供的多个分析角度，而不局限于三张表核心科目。关注分部报告、管理层讨论与分析、报表附注、财务摘要中披露的额外信息
3. 选择指标的唯一标准是：**这个数据点能否支撑一段有价值的分析**——能用来做趋势判断、同行对比、或风险识别

**清单深度校验**：

回顾当前清单，回答：基于这些指标，我能写出几页有实质内容的分析？每个指标对应报告中的哪个分析角度？如果某个分析角度完全缺乏数据支撑，说明报表本身未披露，标记为后续报告中的数据局限。

指标数量不是目标。15个指标能写10页就是好清单；30个指标只能堆数字就是坏清单。制定完成后直接进入阶段3。

## 阶段3 — 口径统一与产出

### 阶段3设计原则

本阶段产出的数据将被后续分析反复使用，需遵循两个原则：

1. **可溯源**：每个数据点的备注中应包含足够信息，让人能在30秒内翻到对应PDF位置找到原文。
2. **可分析**：Excel输出的主表是一张展平的数据表，导入pandas或透视表后无需任何布局调整即可开始分析。不要把数据拆成多个Sheet做"排版"——那是Word的活，不是Excel的。

### 阶段3a — 预置TSV模板

在启动子Agent之前，先用 Write 工具为每家公司×每年创建TSV模板文件。每个模板内容为：

```
指标名称\t数据\t备注
营业收入\t-\t
营业成本\t-\t
...（阶段2d确定的全部指标，每行一个）
```

指标名称与阶段2d清单完全一致，不含单位后缀。数据和备注列预填"-"。模板文件放在 `{output_dir}/tsv/` 目录下。

### 阶段3b — 子Agent提取（并行，批量分组）

将公司按 2-3 家一组分批。同市场放一个Batch——A股和港股混在一起子Agent容易用错关键词。

使用 `Task` 工具并行启动，subagent_type 用 `general`。

**子Agent Prompt 模板**：

```
你是财务数据提取助手。从多家公司的年报TXT中按指定指标清单提取数值，填充到预置的TSV模板中。

=== 市场适配 ===
{本Batch所有公司属同一市场：A股/港股。阶段2c已确认的核心科目在该市场的实际名称：
  - 营业收入 → {实际科目名}
  - 营业成本 → {实际科目名}
  ... 5-8个关键科目即可}

=== 任务清单 ===
{逐项：公司名、股票代码、年份、TXT路径、TSV模板路径}
TSV模板已预置好第一列（指标名称）和表头，你只需填充第二列（数据）和第三列（备注）。
严禁修改第一列的指标名称。

=== 指标清单（与TSV模板第一列一致）===
{从阶段2d确定的完整指标清单}

=== 搜索策略（自主决定路径）===
你可以根据材料的实际结构自行决定搜索顺序——财务摘要、管理层讨论与分析、
完整财务报表、报表附注等，你觉得哪里最可能有数据就从哪里开始。

=== 缺失项验收标准 ===
将一个指标标记为"-"之前，你必须能回答"我做了哪些努力还是没找到"。
每一项缺失至少满足以下2条（越多越可信）：
(1) 用至少3个关键词变体（含繁体/英文）在整个文档中Grep过
(2) 从目录定位到该指标应在的章节，Read确认该处不包含此数据
(3) 财务摘要、MD&A、完整财务报表三处均已检查
(4) 同类科目存在但本项确实不适用（如公司不披露"核心净利润"）

严禁仅因"摘要未披露"就标记为缺失。

=== 两源交叉确认（以下8个指标强制）===
营业收入、毛利、归母净利润、总资产、归母权益、有息负债（总借贷）、
经营活动现金流、合约销售额 —— 每个指标至少从两个独立来源核对。

=== 返回前自检 ===
在报告"完成"之前，必须运行一遍以下自检并在最终消息中输出结果：
1. 指标总数: __  已提取: __  缺失: __
2. 每个缺失项逐个说明：[指标名] → 搜索了X/Y/Z关键词变体，检查了A/B/C章节，确认未披露
3. 是否有因单位换算困难而跳过的指标？（如有，列出原始值和单位）
4. 是否有口径差异？（同一科目在不同公司/年份的名称不统一，列出差异）

=== 填充规则 ===
- 数据: 统一单位(亿元/%)，数值+单位写在一起如"2,435亿"
- 单位换算: 原始单位若是"千元""百万元""十亿元"，必须换算为"亿元"再填入；
  不要因换算困难跳过——原始值和单位写在备注中
- 备注: 注明数据所在章节/页数、原始科目名、原始值和单位，便于30秒内定位原文

完成后仅返回自检结果摘要。严禁返回TSV或TXT原文内容。
```

**并行执行**：在单个消息中同时启动所有批次的子Agent调用。

### 阶段3c — 处理口径差异

汇总各子Agent报告的口径差异，自行判断并统一：
- 同一指标在不同公司的科目名称映射
- 发现口径差异较大无法自行判断时再向用户确认，否则直接更新TSV并继续。

### 阶段3d — 复核

读取各公司TSV文件，逐公司复核数据一致性：
- 交叉验证（如资产=负债+权益的勾稽关系）
- 如涉及多年份，检查同比变化是否合理
- 发现异常自行纠正，汇总缺失项和口径差异。

### 阶段3e — 产出Excel

编写 JSON 配置文件，然后调用预置脚本生成 **一个** Excel 文件（`数据底稿.xlsx`），第一个 Sheet 为汇总透视表，后续为各公司明细。

**Step 1** — 创建 `{output_dir}/excel_config.json`：

```json
{
  "companies": [
    {
      "name": "万科",
      "periods": { "2024": "{output_dir}/tsv/万科_2024.tsv", "2025": "{output_dir}/tsv/万科_2025.tsv" }
    },
    {
      "name": "金茂",
      "periods": { "2024": "{output_dir}/tsv/金茂_2024.tsv", "2025": "{output_dir}/tsv/金茂_2025.tsv" }
    }
  ],
  "indicators": ["营业收入", "营业成本", "毛利", "毛利率", ...],
  "output": "{output_dir}/数据底稿.xlsx"
}
```

- `companies`：按用户指定的公司顺序排列。`name` 为公司简称，`periods` 为期间标签→TSV路径的映射（key 可以是年度"2024"、季度"202403"、半年度"2024H1"等任意字符串）
- `indicators`：指标显示顺序，先绝对值后比率；未在清单中但 TSV 里存在的指标会自动追加到末尾

**Step 2** — 调用脚本：

```bash
python {SD}/generate_excel.py --config "{output_dir}/excel_config.json"
```

脚本自动生成 `数据底稿.xlsx`，结构：Sheet 1 = 汇总透视表（指标 × 公司-期间），后续 Sheet = 各公司各期间明细（名称 / 数据 / 备注）。

## 阶段4 — 分析报告

数据提取和 Excel 产出完成后进入本阶段，依次产出：汇总表 → 报告大纲 → HTML 报告。

### 阶段4a — 输出 Markdown 汇总表

在对话中输出一张 **指标 × 公司-期间** 交叉透视表，严格按以下模板：

```
|               | 单位 | 万科       | 万科       | 金茂       | 金茂       |
|               |      | 2024年     | 2025年     | 2024年     | 2025年     |
|---------------|------|------------|------------|------------|------------|
| 投资性房地产   | 亿   | 1,200      | 1,350      | 880        | 920        |
| 总负债         | 亿   | 8,500      | 9,200      | 6,100      | 6,800      |
| 毛利率         | %    | 16.0       | 14.2       | 18.5       | 15.5       |
```

规则：
1. **列结构**：第1列指标名，第2列单位，其余按"公司A YYYY年 | …"排列。第1行表头为公司简称，第2行为年份
2. **列顺序**：按用户指定的公司顺序，每家公司内期间升序
3. **行顺序**：先绝对值/规模类，再比率/计算类
4. **值格式**：数据单元格只写纯数字（不加单位），保留适当小数位；NA 填 `-`
5. **禁止**：不输出数据来源列、备注列、TSV 原文或任何调试信息

### 阶段4b — 制作分析报告大纲

基于汇总数据，列出分析报告大纲（各级标题即可），内容结构由你根据本次数据和公司特点自行发挥。

**大纲深度自检**（列出后、确认前自查）：

1. **章节数量**：标准深度报告至少 8 个分析章节（含行业背景）。如果 ≤6 章，说明指标提取覆盖面不够——回TXT搜索补充数据。
2. **图表覆盖率**：每个分析章节至少对应 1 张图表或 1 张数据表格。如果某个章节既无图表也无表格，要么补数据，要么合并到其他章节。
3. **篇幅估算**：每章按图/表 + 3段分析文字 ≈ 1-1.5 印页估算。总章节数是否满足阶段1确定的报告目标篇幅？
4. **盲区检查**：报告中是否有理应涉及但因数据缺失而完全跳过的分析角度？如有，要么回阶段2d补指标，要么在报告中标注"数据局限"。

不通过自检则扩展大纲，通过后列出一次性向用户确认。

### 阶段4c — 生成 HTML 分析报告

撰写 HTML 报告，写入 `{output_dir}/分析报告.html`：

- 使用 Chart.js (CDN) 渲染图表，Chart.js 配置直接在 `<script>` 标签中手写——图表数据来自阶段3e产出的 Excel 数据底稿
- 按大纲组织章节，每节包含分析段落 + 对应图表
- CSS 样式内嵌，简洁专业，适合打印
- KPI 卡片和表格数值必须来自 Excel，**禁止编造数字**
- 分析段落的文字内容由你撰写

**每节深度要求**（标准/深度报告）：

- 每个分析章节至少包含：1 段 KPI 卡片或总结句 + 1 张图表或 1 张数据表格 + 3 段以上分析文字（趋势解读、同行对比、原因分析）
- 报告开头应有 KPI 概览卡片（3-6 个关键数字，形成第一印象）
- 报告结尾应有综合评价/分层结论/风险提示，不能只有数据没有判断
- 对于数据缺失的维度，在相关章节末尾用简短说明标注，而非完全跳过
- 可补充 `generate_charts.py` 未覆盖的自定义图表（在 `<script>` 中手写 Chart.js 配置），确保每个重要维度都有可视化呈现

## 规则

- 只从官方披露渠道下载，原始PDF必须保留
- 每个数据点可追溯到报告具体位置
- 确定性脚本默认使用低噪音参数（如 `--quiet`），禁止把下载明细、PDF解析警告、TXT正文片段作为工具输出刷入主会话；需要诊断时再临时开启详细输出
- 子Agent绝不返回原始文件内容（txt正文、TSV数据），只返回路径和摘要
- 子Agent遇到无法解决的问题时标记清楚，不编造数据

### 子Agent数据质量原则

1. **不规定搜索路径**：每个文档的结构不同（年报/中报/通函/招股书），子Agent自行判断从何处开始搜索最高效
2. **规定验收标准**：标记缺失项之前必须穷举关键词变体、检查所有可能位置；禁止仅因"摘要未披露"跳过
3. **关键指标双源确认**：核心财务科目（收入/毛利/归母净利润/总资产/权益/有息负债/经营现金流/合约销售额）必须从两个独立来源交叉核对
4. **自检清单**：子Agent返回数据前必须输出自检结果（提取数/缺失项举证/换算困难/口径差异），每个缺失项需写清搜索了什么、找不到的原因
